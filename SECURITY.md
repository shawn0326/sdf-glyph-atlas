# Security Policy / 安全策略

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's **Report a vulnerability** private security advisory form for this repository and include affected versions, impact, reproduction steps, and any suggested mitigation. You should receive an initial response within seven days.

Only the latest published minor line receives security fixes while the project is below 1.0. A confirmed fix will be released as soon as practical and credited according to the reporter's preference.

## 报告安全问题

怀疑存在漏洞时，请不要创建公开 Issue。请使用本仓库 GitHub **Report a vulnerability** 私有安全公告表单，并提供受影响版本、影响、复现步骤和建议的缓解方案。项目会尽量在七天内首次回复。

项目处于 1.0 之前时，仅最新发布的 minor 系列会接收安全修复。确认问题后会尽快发布修复，并按报告者意愿致谢。
